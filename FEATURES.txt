17. PRIVATE VAULT
────────────────────────────────────────────────────────────────────────────────
  A fully isolated, encrypted section of the user's library for media they want
  to keep completely hidden from their public profile, activity feed, stats,
  followers, and even normal app views. The vault is locked by default and must
  be explicitly unlocked each session.

  WHAT THE VAULT STORES:
  - A separate hidden library (same media types: anime, manga, movies, etc.)
  - Vault entries NEVER appear in:
      • Public profile / profile stats
      • Activity feed (own or others')
      • Global statistics & analytics
      • Custom lists (unless vault list)
      • Search suggestions or recommendations shown to others
  - Vault has its own independent stats page (watch time, ratings, etc.)
  - Vault has its own Season Continuation & Watch Timeline (fully separate)

  AUTHENTICATION — THREE UNLOCK METHODS:

  1. VAULT PIN
     - Separate 6-digit PIN, completely different from account login PIN
     - Set during vault first-time setup
     - Wrong PIN ×5 → vault locks for a cooldown period (e.g. 15 minutes)
     - PIN can be changed only after successful vault unlock

  2. VAULT OTP
     - One-time password sent to registered email or authenticator app
     - Completely separate OTP flow from the main account login OTP
     - Used as a fallback if PIN is forgotten or biometric fails
     - OTP expires in 5 minutes

  3. BIOMETRIC UNLOCK (via Mobile QR Bridge)
     - Designed for desktop sessions where the browser has no biometric access
     - Flow:
         Step 1 → User clicks "Unlock with Fingerprint" on desktop
         Step 2 → A unique time-limited QR code is displayed on screen
                  (expires in 60 seconds, single-use)
         Step 3 → User scans QR code with NeonVault mobile app
         Step 4 → Mobile app prompts fingerprint / Face ID scan
         Step 5a → If biometric SUCCEEDS:
                     Mobile sends a signed token to the server
                     Desktop session receives unlock confirmation in real-time
                     Vault is unlocked on desktop
         Step 5b → If biometric FAILS (×3 attempts):
                     Mobile shows failure message
                     Desktop falls back to Vault PIN or Vault OTP prompt
     - Biometric bridge uses a short-lived encrypted token (never stores biometric data)
     - QR code is single-use and invalidated immediately after scan
     - Each vault unlock session has a timeout (auto-locks after inactivity)

  VAULT SESSION MANAGEMENT:
  - Auto-lock after configurable inactivity timeout (5 / 15 / 30 min or manual)
  - "Lock Vault Now" button always visible when vault is open
  - Vault locked/unlocked status is never exposed to other users or logs
  - Failed unlock attempts silently logged (visible only inside vault settings)
  - Vault can be fully wiped/reset via email confirmation (emergency reset)

  VAULT SETTINGS (accessible inside vault only):
  - Change Vault PIN
  - Enable / Disable biometric unlock
  - Set inactivity auto-lock timer
  - View failed unlock attempt log
  - Export vault data (encrypted backup)
  - Emergency vault reset (sends confirmation email to registered address)
