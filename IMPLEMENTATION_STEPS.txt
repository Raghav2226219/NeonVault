────────────────────────────────────────────────────────────────────────────────
PHASE 1 — PROJECT SETUP & FOUNDATION
────────────────────────────────────────────────────────────────────────────────

STEP 1 — Initialize Backend
  [ ] cd backend && npm init -y
  [ ] Install dependencies:
        npm install express cors dotenv bcryptjs jsonwebtoken
                    @prisma/client nodemailer express-rate-limit
                    cookie-parser helmet morgan uuid
  [ ] Install dev dependencies:
        npm install -D prisma nodemon
  [ ] Create server.js — main Express app entry point
  [ ] Set up .env with:
        PORT
        DATABASE_URL           (Neon PostgreSQL connection string)
        JWT_SECRET
        JWT_REFRESH_SECRET
        EMAIL_USER
        EMAIL_PASS
        CLIENT_URL
        VAULT_ENCRYPTION_KEY
  [ ] Set up .gitignore (node_modules, .env, dist)
  [ ] Add "dev": "nodemon server.js" to package.json scripts

STEP 2 — Initialize Frontend
  [ ] cd frontend
  [ ] npx create-vite@latest . -- --template react
  [ ] Install dependencies:
        npm install axios react-router-dom react-query
                    zustand framer-motion recharts
                    react-hot-toast qrcode.react
  [ ] Set up .env with:
        VITE_API_URL=http://localhost:5000/api
        VITE_JIKAN_URL=https://api.jikan.moe/v4
        VITE_TMDB_URL=https://api.themoviedb.org/3
        VITE_TMDB_KEY=<your_key>
        VITE_COMICK_URL=https://api.comick.io
  [ ] Set up vite.config.js with proxy to backend
  [ ] Set up .gitignore (node_modules, dist, .env)

STEP 3 — Database Setup (Neon + Prisma)
  [ ] Create a Neon project at neon.tech
  [ ] Copy the DATABASE_URL into backend/.env
  [ ] Run: npx prisma init (inside backend/)
  [ ] Define Prisma schema (see STEP 3 details below)
  [ ] Run: npx prisma migrate dev --name init
  [ ] Run: npx prisma generate
  [ ] Create backend/config/db.js — PrismaClient singleton

  PRISMA SCHEMA MODELS:
    • User              — id, email, username, password, avatar, bio,
                          isPublic, createdAt, refreshToken
    • Library           — id, userId, mediaId, mediaType, status,
                          currentEp, currentChapter, totalEp, totalChapters,
                          score, notes, startDate, endDate, isFavourite,
                          rewatchCount, createdAt, updatedAt
    • VaultEntry        — id, userId, mediaId, mediaType, status,
                          currentEp, currentChapter, score, notes,
                          createdAt, updatedAt
    • VaultAuth         — id, userId, pinHash, otpSecret, biometricEnabled,
                          failedAttempts, lockedUntil
    • VaultSession      — id, userId, token, expiresAt, createdAt
    • BiometricQR       — id, userId, qrToken, expiresAt, used
    • CustomList        — id, userId, name, description, isPublic, createdAt
    • CustomListItem    — id, listId, mediaId, mediaType
    • Review            — id, userId, mediaId, mediaType, content,
                          score, spoiler, createdAt
    • Comment           — id, userId, mediaId, mediaType, content,
                          parentId, createdAt
    • Follow            — id, followerId, followingId, createdAt
    • Notification      — id, userId, type, message, read, createdAt
    • ActivityLog       — id, userId, action, mediaId, mediaType,
                          detail, createdAt
    • WatchedEpisode    — id, userId, mediaId, season, episode,
                          watchedAt, isVault
